## 1. はじめに

本マニュアルは、ドリンク在庫管理システムの管理者向け機能を解説します。管理者は一般利用者の機能に加え、マスタ管理・入庫・設定変更等の管理機能を使用できます。

### 管理者アカウント

- 社員マスタで権限が **「管理者」** に設定されたアカウントのみ管理画面にアクセスできます
- 初期管理者アカウント: 社員コード `00000001`（管理者）

---

## 2. 管理画面へのアクセス

1. 管理者アカウントでログインします
2. ドリンク一覧画面のヘッダーに **「管理」** ボタンが表示されます
3. ボタンを押すと管理ダッシュボードに遷移します

### 管理画面メニュー（サイドバー）

| メニュー | 機能 |
|----------|------|
| ダッシュボード | 在庫状況・消費傾向のサマリー |
| 入庫 | ドリンクの入庫処理 |
| 棚卸し | 実在庫との照合 |
| ドリンクマスタ | ドリンク種類の管理 |
| 社員マスタ | 社員情報の管理 |
| 履歴 | 取引・棚卸し履歴の閲覧 |
| 設定 | システム全般の設定 |

---

## 3. ダッシュボード

管理者ダッシュボードでは、在庫状況をリアルタイムに把握できます。

### 表示項目

| 項目 | 説明 |
|------|------|
| 本日の消費数 | 当日の取り出し合計数 |
| 総在庫数 | 全ドリンクの在庫合計 |
| 在庫状況グラフ | ドリンクごとの在庫数を横棒グラフで表示（色分け：十分=青 / 少=オレンジ） |
| 週間推移 | 直近7日間の消費数を日別棒グラフで表示 |
| 本日の内訳 | ドリンクごとの当日消費数（バッジ表示） |
| 直近のアラート | 棚卸し差分の直近記録（赤=大きな差分 / 黄=小さな差分） |

## 4. 入庫管理

ドリンクを補充した際に在庫数を増やす機能です。

### 手順

1. サイドバーから **「入庫」** を選択します
2. **ドリンク** をドロップダウンから選択します（現在の在庫数が表示されます）
3. **数量** を入力します（1〜999の範囲、+/−ボタンまたは直接入力）
4. 入庫後の在庫数がプレビュー表示されます（例: `現在: 5 → 入庫後: 15`）
5. **「入庫」** ボタンを押します
6. 画面下部の直近履歴テーブルに記録が表示されます

> **注意:** 入庫操作は管理者のみ実行できます。入庫した社員名が自動で記録されます。

---

## 5. 棚卸し

システムの理論在庫と実際の在庫数を照合し、差分を記録・補正する機能です。

### 手順

1. サイドバーから **「棚卸し」** を選択します
2. 全ドリンクの一覧テーブルが表示されます
   - **ドリンク名**: ドリンクの名称
   - **理論在庫**: システム上の現在在庫数（変更不可）
   - **実在庫**: 実際に数えた在庫数を入力（0以上の整数）
   - **差分**: 自動計算（実在庫 − 理論在庫）
3. すべてのドリンクの実在庫を入力します
4. **「確定」** ボタンを押します
5. 結果画面に以下が表示されます：
   - 差分の有無
   - 通知送信の有無
   - 在庫更新の完了

### 棚卸し確定時の動作

- 全ドリンクのシステム在庫が **実在庫の値に更新** されます
- 差分がある場合、Webhook通知が送信されます（設定済みの場合）
- 棚卸し結果は履歴に記録されます

### データ保持期間

- 棚卸し記録は **1年間** 保持されます
- 1年以上経過した記録は、管理画面からクリーンアップ（削除）できます

## 6. ドリンクマスタ管理

ドリンクの種類を管理します。

### ドリンク一覧

サイドバーから **「ドリンクマスタ」** を選択すると、登録済みドリンクの一覧が表示されます。

| 列 | 説明 |
|----|------|
| 表示順 | 画面上の並び順（昇順） |
| ドリンク名 | ドリンクの名称 |
| 在庫数 | 現在のシステム在庫 |
| ステータス | 有効 / 無効 |
| 操作 | 編集・削除ボタン |

### ドリンクの新規登録

1. **「新規追加」** ボタンを押します
2. 以下を入力します：
   - **ドリンク名**（必須・1〜100文字）
   - **表示順**（任意・整数）
   - **初期在庫数**（任意・0以上の整数）
3. **「保存」** ボタンを押します

### ドリンクの編集

1. 一覧の **「編集」** ボタンを押します
2. ドリンク名・表示順・有効/無効を変更できます
3. **「保存」** ボタンを押します

> **注意:** 在庫数は編集画面からは変更できません。在庫の調整は入庫または棚卸しで行ってください。

### ドリンクの削除（無効化）

1. 一覧の **「削除」** ボタンを押します
2. 確認ダイアログで **「OK」** を押します
3. ドリンクは物理削除されず、**無効（非表示）** になります

> **補足:** 無効化されたドリンクは取り出し・返却画面に表示されなくなりますが、過去の履歴データは保持されます。

### ドリンク画像

- ドリンクには画像を設定できます
- 対応形式: **JPEG、PNG、WebP**
- ファイルサイズ上限: **2MB**

## 7. 社員マスタ管理

システムを利用する社員情報を管理します。

### 社員一覧

サイドバーから **「社員マスタ」** を選択すると、登録済み社員の一覧が表示されます。

| 列 | 説明 |
|----|------|
| 社員コード | ログインに使う一意のコード（最大8桁） |
| 氏名 | 社員名 |
| 権限 | 管理者 / 一般 |
| ステータス | 有効 / 無効 |
| 操作 | 編集・削除ボタン |

### 社員の新規登録

1. **「新規追加」** ボタンを押します
2. 以下を入力します：
   - **社員コード**（必須・1〜8文字・重複不可）
   - **氏名**（必須・1〜100文字）
   - **権限**（管理者 / 一般、デフォルト：一般）
3. **「保存」** ボタンを押します

### 社員の編集

1. 一覧の **「編集」** ボタンを押します
2. **氏名** と **権限** を変更できます
3. **「保存」** ボタンを押します

> **注意:** 社員コードは変更できません。変更が必要な場合は、当該社員を無効化し新しいコードで再登録してください。

### 社員の削除（無効化）

1. 一覧の **「削除」** ボタンを押します
2. 確認ダイアログで **「OK」** を押します
3. 社員は物理削除されず、**無効（ログイン不可）** になります

### CSV一括インポート

大量の社員を一括で登録・更新できます。

#### CSVフォーマット

```csv
社員番号,氏名,権限
00000001,山田太郎,管理者
00000002,鈴木花子,一般
```

| 列名 | 説明 | 必須 |
|------|------|------|
| 社員番号 | 社員コード（最大8文字） | 必須 |
| 氏名 | 社員の名前 | 必須 |
| 権限 | 「管理者」または「一般」 | 必須 |

#### インポート手順

1. **「テンプレートダウンロード」** ボタンでCSVテンプレートを取得します
2. テンプレートに社員情報を入力します
3. **「CSVインポート」** ボタンを押し、ファイルを選択します
4. インポート結果が表示されます：
   - **作成**: 新規登録された社員数
   - **更新**: 既存情報が更新された社員数
   - **スキップ**: 処理対象外の件数
   - **エラー**: 登録に失敗した件数

> **補足:** 既に存在する社員コードの場合、氏名と権限が上書き更新されます。

## 8. 履歴管理

取引履歴と棚卸し履歴を閲覧できます。

### アクセス方法

サイドバーから **「履歴」** を選択します。

### タブ切替

| タブ | 内容 |
|------|------|
| 取引履歴 | 取り出し・返却の記録 |
| 棚卸し履歴 | 棚卸し結果の記録 |

### 取引履歴

#### 表示項目

| 列 | 説明 |
|----|------|
| 日時 | 取引日時 |
| 種別 | 取り出し / 返却 |
| 社員番号 | 操作した社員のコード |
| 社員名 | 操作した社員の名前 |
| ドリンク名 | 対象のドリンク |
| 数量 | 取引数量 |
| お客様名 | お客様名（取り出し時に入力された場合） |

#### 絞り込み

- **期間**: 開始日〜終了日
- **ドリンク**: ドリンク種類で絞り込み
- **種別**: 取り出し / 返却

### 棚卸し履歴

#### セッション表示

棚卸しは実施者・日時でグループ化（セッション）され、展開して詳細を確認できます。

#### 詳細表示項目

| 列 | 説明 |
|----|------|
| 日時 | 棚卸し実施日時 |
| 実施者番号 | 実施者の社員コード |
| 実施者名 | 実施者の名前 |
| ドリンク名 | 対象ドリンク |
| 理論在庫 | 棚卸し時のシステム在庫 |
| 実在庫 | 実際にカウントした数 |
| 差分 | 実在庫 − 理論在庫 |

### ページネーション

- 1ページあたり **50件** 表示
- ページ送りボタンで前後のページに移動できます

## 9. 設定

システム全体の動作を制御する設定項目です。

### アクセス方法

サイドバーから **「設定」** を選択します。

### 設定項目一覧

| 設定項目 | 説明 | デフォルト値 |
|----------|------|-------------|
| Webhook URL | 通知の送信先URL（Google Chat等） | 空（未設定） |
| 在庫低下閾値 | この本数以下で在庫低下通知を送信 | 3 |
| 棚卸しリマインダー時刻 | 棚卸しリマインダーの時刻（HH:MM形式、カンマ区切り） | 09:00,18:00 |
| ロックアウト最大試行回数 | ログイン連続失敗でロックされるまでの回数 | 3 |
| ロックアウト期間（分） | ロックアウトの解除までの時間 | 5 |
| セッションタイムアウト（分） | 無操作でログアウトされるまでの時間 | 30 |

### 保存

- すべての設定を変更後、**「保存」** ボタンを押すと一括で反映されます

## 10. Webhook通知

外部チャットツール（Google Chat等）にアラートを自動送信する機能です。

### 通知が送信される条件

通知は以下の **2つのケース** で送信されます。

#### 1. 棚卸し時に差分がある場合

棚卸しを実行し、理論在庫と実在庫に1つでも差分がある場合に通知が送信されます。

**通知フォーマット:**

```
⚠️ 棚卸し差分アラート
実施者: 山田太郎（00000001）
実施日時: 2026/2/13 15:00:00

差分あり:
• コーヒー（ブラック）: 理論 50 → 実数 48（-2本）
• お茶（緑茶）: 理論 30 → 実数 32（+2本）

差分合計: ±4本
```

#### 2. 取り出し時に在庫が閾値以下になった場合

取り出し操作後、対象ドリンクの在庫が「在庫低下閾値」（デフォルト: 3本）以下になった場合に通知が送信されます。

**通知フォーマット（在庫低下）:**

```
⚠️ 在庫低下アラート
日時: 2026/2/13 14:30:00

ドリンク: コーヒー（ブラック）
残り 2本（閾値: 3本）

補充をお願いします。
```

**通知フォーマット（在庫切れ）:**

```
🚨 在庫低下アラート
日時: 2026/2/13 14:30:00

ドリンク: コーヒー（ブラック）
在庫切れ

補充をお願いします。
```

### 通知が送信されないケース

- 設定画面で **Webhook URL が空**（未設定）の場合
- **返却操作** では在庫低下通知は送信されません
- 棚卸しで **差分が一切ない** 場合

### Webhook URLの設定方法

1. Google Chat等でWebhook URLを取得します
2. 管理画面の **「設定」** を開きます
3. **「Webhook URL」** 欄にURLを貼り付けます
4. **「保存」** ボタンを押します

## 11. データエクスポート

### 取引履歴のCSVエクスポート

1. **「履歴」** 画面で「取引履歴」タブを選択します
2. 必要に応じてフィルタ条件を設定します（期間・ドリンク・種別）
3. **「CSVエクスポート」** ボタンを押します
4. CSVファイルがダウンロードされます

**エクスポート項目:**

```csv
日時,種別,社員番号,社員名,ドリンク名,数量,お客様名
```

### 棚卸し履歴のCSVエクスポート

1. **「履歴」** 画面で「棚卸し履歴」タブを選択します
2. 必要に応じてフィルタ条件を設定します（期間）
3. **「CSVエクスポート」** ボタンを押します
4. CSVファイルがダウンロードされます

**エクスポート項目:**

```csv
日時,実施者番号,実施者名,ドリンク名,理論在庫,実在庫,差分
```

> **補足:** CSVファイルはUTF-8（BOM付き）で出力されるため、Excelで直接開いて文字化けしません。日時は日本時間（Asia/Tokyo）で出力されます。

## 12. 運用ガイド

### 日常運用

| 作業 | 頻度 | 担当 |
|------|------|------|
| ドリンク取り出し・返却 | 随時 | 全社員 |
| 在庫確認（ダッシュボード） | 毎日 | 管理者 |
| 入庫処理 | ドリンク補充時 | 管理者 |
| 棚卸し | 毎日（推奨：設定のリマインダー時刻） | 管理者または担当者 |

### 定期メンテナンス

| 作業 | 頻度 | 手順 |
|------|------|------|
| 社員マスタ更新 | 入退社時 | 社員マスタから追加/無効化 |
| ドリンクマスタ更新 | ラインナップ変更時 | ドリンクマスタから追加/無効化 |
| 古い棚卸しデータの削除 | 年1回程度 | 棚卸し記録のクリーンアップ |
| Webhook URL確認 | チャットツール変更時 | 設定画面で更新 |

### トラブルシューティング

| 症状 | 原因 | 対処法 |
|------|------|--------|
| ログインできない | 社員コードが無効化されている | 管理者が社員マスタで有効に戻す |
| ログインできない | ロックアウト中 | 5分間（設定値）待つ |
| 在庫数がずれている | 取り出し/返却の記録漏れ | 棚卸しで実在庫に補正する |
| Webhook通知が届かない | URLが未設定 or 無効 | 設定画面でURL確認 |
| 管理画面が表示されない | 権限が「一般」になっている | 他の管理者に権限変更を依頼 |

### セキュリティに関する注意

- 管理者権限は必要最小限の社員にのみ付与してください
- 社員が退職した場合は速やかにアカウントを無効化してください
- Webhook URLは外部に漏洩しないよう管理してください
